# AutoAgent Lessons

Lessons are the bridge between the Python optimization workshop and the TypeScript production app.

## What is a Lesson?

A Lesson is a structured document that captures a winning meta-agent optimization and describes how to translate it to production TypeScript. Every time the meta-agent finds an improvement that beats the baseline (status=keep in `results.tsv` with a meaningful score delta), a Lesson document is generated.

## Where do Lessons live?

```
autoagent/lessons/
  LESSON_TEMPLATE.md    # Fillable skeleton for new lessons
  README.md             # This file
  LESSON-001.md         # First lesson (example)
  LESSON-002.md         # Second lesson
  ...
```

## Approval Statuses

| Status | Meaning |
|--------|---------|
| **DRAFT** | Generated by the lesson extractor, not yet reviewed |
| **REVIEWED** | Human has read and validated the lesson |
| **APPROVED** | Human has approved the TypeScript translation and preview test |
| **APPLIED** | Merged to production, applied_commit hash recorded |
| **REJECTED** | Human decided not to apply this lesson (reason documented) |

## Generating Lessons

Run the lesson extractor after an optimization session:

```bash
python autoagent/scripts/extract_lessons.py
```

Options:
- `--threshold 0.02` — minimum score delta to generate a lesson (default: 0.02)
- `--limit 5` — maximum number of lessons to generate per run

## Key Rules

1. **Lessons are read-only after APPLIED** — do not edit applied lessons
2. **One lesson per winning experiment** — the extractor is idempotent
3. **Human gates are mandatory** — no automated path from DRAFT to APPLIED
4. **Architecture constraints always apply** — lessons that violate DA-UC-001 rules are auto-rejected
